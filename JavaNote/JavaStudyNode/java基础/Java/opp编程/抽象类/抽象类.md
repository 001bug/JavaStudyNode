这里有个问题,父类方法的不确定性
当父类的某些方法, 需要声明, 但是又不确定如何实现, 可以将其声明为[抽象方法], 那么这个类就是抽象类
而抽象方法就是没有实现的方法,所谓没有实现就是指,没有方法体

==语法== 当父类的一些方法不能确定时,可以用abstract关键字来修饰该方法, 这个方法就是抽象方法,用abstract 来修饰该类就是抽象类
```
public abstract Cat{
	private String name;

	public Animal(String name){
		this.name=name;	
	}

	public abstract void eat();
}
```


抽象类详解
----

**抽象类的介绍**
1. 用abstract 关键字来修饰一个类时, 这个类就叫抽象类  
     访问修饰符 abstract 类名{ 
    } 
2. 用abstract 关键字来修饰一个方法时,这个方法就是抽象方法(抽象方法没有方法体)
3. 抽象类的价值更多作用是在于设计, 是设计者设计好后, 让子类继承实现抽象类()
4. 抽象类, 面试常出, 在框架和设计模式使用较多

**抽象类的使用细节**
1. 抽象类不能被实例化
2. 抽象类不一定要包含abstract方法,一旦类包含了abstract方法,则这个类必须声明为abstract
3. ==abstract只能修饰类和方法==, 不能修饰属性和其他
4. 抽象类可以有任意成员 [抽象类还是类]  ,成员比如(非抽象方法,构造器,静态属性)
5. 抽象方法不能有主体,即不能实现,如下
    ![[Pasted image 20240219202048.png]]
6. 如果一个类[继承]了抽象类, 则它必须[实现]抽象类的所有抽象方法, 除非它自己也声明为abstract类
 ![[Pasted image 20240219203157.png]]
7. 抽象方法不能使用private,final和static来修饰, 因为这些关键字都是和[重写]相违背的
